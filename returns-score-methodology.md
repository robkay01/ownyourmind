# Returns Score Methodology

How we assess whether a DeAI token captures value from its protocol. Five dimensions, scored criteria, and why the exact number matters less than the reasoning behind it.

---

## The Dual-Score Philosophy

**Freedom AND Returns.** That is the thesis of this site, and it requires two independent measures:

| Score | What It Measures | For Whom |
|-------|------------------|----------|
| **Freedom Score** | How decentralized is the protocol? | Sovereignty seekers, censorship resistors, builders who need neutral infrastructure |
| **Returns Score** | Does the token capture value? | Investors, participants, anyone allocating capital |

A project can score highly on one dimension and poorly on the other. A fair-launch network with perfect distribution but no revenue mechanism. A revenue-generating platform owned by a single company. Both are real patterns in DeAI.

Neither score is "better." They answer different questions for different audiences. This methodology explains how we calculate the Returns Score.

---

## What Returns Score Assesses

Returns Score measures whether the token has a credible path to value appreciation based on protocol activity. It does NOT measure:

- Whether the price will go up (we have no idea)
- Whether the team is honest (that is partially covered by Freedom Score)
- Whether the technology works (that is covered in project reviews)

It DOES measure:

- Is there a mechanical link between usage and token value?
- Is the supply schedule sustainable?
- Can the protocol generate real revenue?
- Can holders actually enter and exit positions?

A high Returns Score does not guarantee returns. A low Returns Score does not guarantee losses. It assesses structural alignment, not market outcomes.

---

## The Five Dimensions

Returns Score is calculated across five dimensions, weighted by importance:

| Dimension | Weight |
|-----------|--------|
| Token Utility | 20% |
| Value Accrual | 25% |
| Supply Dynamics | 20% |
| Revenue Sustainability | 20% |
| Liquidity & Access | 15% |
| **Total** | **100%** |

Each dimension is scored from 0 to 20 points. Weighted total produces a score from 0 to 100.

---

## Dimension 1: Token Utility (20%)

**What it measures:** What can holders actually do with the token? A token with multiple genuine utilities is more likely to maintain demand than one that exists solely for speculation.

### Scoring Rubric

| Score | Criteria |
|-------|----------|
| **18-20** | **Essential utility.** Multiple functions: staking for yield, governance rights, access to services, payment currency, collateral for borrowing, required for node operation. Token is structurally essential to use the protocol. |
| **14-17** | **Strong utility.** 2-3 clear utilities. Token has genuine purpose but protocol could function partially without it. Example: governance + staking, but network would run without the token. |
| **10-13** | **Moderate utility.** Single meaningful utility (governance only, or staking emissions only). Token feels like it could be replaced by USDC. |
| **6-9** | **Weak utility.** Utility exists on paper but not in practice. "Governance" with no real decisions to make. "Staking" with no purpose beyond emissions. Token feels bolted on. |
| **0-5** | **No real utility.** Pure speculation vehicle. No reason to hold except price appreciation. |

### What Counts as Utility

- **Payment for services** (inference, compute, storage)
- **Staking for yield** (from real revenue, not just emissions)
- **Staking for access** (required to operate nodes, validators)
- **Governance** (meaningful decisions, not just advisory)
- **Collateral** (for borrowing, slashing, bonding)
- **Access gating** (token required to use the protocol)

### What Does NOT Count

- Airdrops or rewards for holding
- Speculative value ("price will go up")
- Utility on a roadmap that does not exist yet
- Governance over trivial decisions

---

## Dimension 2: Value Accrual (25%)

**What it measures:** Is there a mechanical link between protocol activity and token value? When someone uses the protocol, does value flow to holders?

This is the most important dimension. Many projects have "utility" but no connection between usage and value.

### Scoring Rubric

| Score | Criteria |
|-------|----------|
| **22-25** | **Direct accrual.** Clear mechanical link: fees → buy/burn → holders. Revenue-sharing to stakers. Deflationary pressure from usage. More activity = unambiguously good for holders. |
| **17-21** | **Indirect accrual.** Token needed for access, so demand increases with usage. Staking yields from real revenue (not emissions). Value flows to holders through measurable channels. |
| **12-16** | **Speculative accrual.** Token benefits from network effects and attention, but no direct mechanism. "Price will go up because the network grows" - may be true but not guaranteed mechanically. |
| **6-11** | **Weak accrual.** Token may benefit from future governance decisions or theoretical fee switches. Value accrual is aspirational, not implemented. |
| **0-5** | **No accrual path.** Token has no relationship to protocol success. Protocol could 10x revenue and token holders would see zero benefit. |

### Accrual Mechanisms (Strongest to Weakest)

1. **Fee distribution** - Fees collected and distributed to stakers/token holders
2. **Buy and burn** - Revenue used to buy tokens from market and burn them
3. **Burn on use** - Tokens burned when services are consumed
4. **Staking requirement** - Must stake to use/operate, reducing float
5. **Access gating** - Must hold token to use protocol
6. **Governance rights only** - No direct value capture
7. **None** - No mechanism

---

## Dimension 3: Supply Dynamics (20%)

**What it measures:** Is the supply schedule sustainable, or will holders be diluted into oblivion?

### Scoring Rubric

| Score | Criteria |
|-------|----------|
| **18-20** | **Deflationary or fixed.** Max supply capped (like Bitcoin). Low or decreasing circulating supply. Emissions end or become negligible within 5 years. No significant insider unlocks remaining. |
| **14-17** | **Controlled inflation.** Predictable emission schedule (4+ year halving or decay). Moderate inflation (3-10% annually). Insider tokens fully vested or minimal remaining. |
| **10-13** | **Elevated inflation.** High inflation (10-20% annually). Long emission tail (10+ years). Significant insider allocation but vesting in progress. |
| **6-9** | **Dilutive.** Very high inflation (>20% annually). Major insider unlocks ahead (team, investors, foundation). Emissions far exceed demand. |
| **0-5** | **Supply shock risk.** Infinite supply or runaway inflation. Large token overhang (50%+ of supply locked, unvested, or foundation-controlled). Massive dilution guaranteed. |

### Key Metrics

- **Inflation rate** - Annual % increase in circulating supply
- **Emission schedule** - How long until emissions become negligible?
- **Insider allocation** - % held by team, investors, foundation
- **Vesting status** - Are insider tokens locked or can they dump?
- **Max supply** - Is there a cap, or infinite?

---

## Dimension 4: Revenue Sustainability (20%)

**What it measures:** Does the protocol generate real revenue from paying customers, or is it entirely dependent on token emissions to attract participants?

### Scoring Rubric

| Score | Criteria |
|-------|----------|
| **18-20** | **Product-market fit.** Protocol generates real revenue from external customers (not just token incentives). Revenue covers or exceeds emissions. Sustainable without inflation. |
| **13-17** | **Revenue-generating.** Meaningful revenue exists but does not yet cover emissions. Clear path to sustainability. Real users, not just yield farmers. |
| **8-12** | **Emission-dependent.** Minimal external revenue. Participants are primarily emission farmers. No clear business model yet, but infrastructure is functional. |
| **4-7** | **Speculative.** No revenue. Pure incentive layer. Participants are there for token rewards, not utility. |
| **0-3** | **Extractive or broken.** Protocol loses money on every transaction (paying more in incentives than it earns). No path to sustainability. |

### What Counts as Revenue

- Fees paid by external users for services (compute, inference, data)
- Transaction fees collected by the protocol
- Subscription or SaaS revenue
- Marketplace fees

### What Does NOT Count

- Emissions to participants (that is cost, not revenue)
- Grants or venture funding
- Token sales
- Treasury investment returns

---

## Dimension 5: Liquidity & Access (15%)

**What it measures:** Can holders actually buy and sell the token without destroying themselves with slippage?

### Scoring Rubric

| Score | Criteria |
|-------|----------|
| **13-15** | **Tier-1 liquid.** Listed on Binance, Coinbase, Kraken. High DEX liquidity across multiple chains. Daily volume >$10M. Large holders can exit without moving price. |
| **10-12** | **Tier-2 liquid.** Listed on mid-tier CEXs (Gate, KuCoin, Bybit). Moderate DEX liquidity. Daily volume $1-10M. Reasonable entry/exit for most holders. |
| **6-9** | **Thin liquidity.** DEX-only or low-tier CEX. Daily volume $100K-$1M. Significant slippage on moderate trades ($50K+). Entry/exit requires planning. |
| **0-5** | **Illiquid.** Minimal liquidity. Daily volume <$100K. Large holders cannot exit without massive slippage. Functionally trapped. |

### Key Metrics

- **CEX listings** - Which exchanges carry the token?
- **DEX liquidity** - Total liquidity in major pools
- **Daily volume** - How much trades per day?
- **Market cap to volume ratio** - Lower = more liquid

---

## Grade Bands

The weighted total produces a score from 0 to 100, mapped to letter grades:

| Score | Grade | Interpretation |
|-------|-------|----------------|
| 90-100 | **A** | Exceptional value capture. Token is structurally aligned with protocol success. |
| 75-89 | **B** | Strong fundamentals. Minor concerns but credible accrual path. |
| 60-74 | **C** | Adequate. Some value capture but significant gaps or risks. |
| 45-59 | **D** | Weak. Token economics are misaligned or unsustainable. |
| 0-44 | **F** | Failing. Little to no connection between protocol success and token value. |

## Limitations

Returns Score has important limitations:

1. **Point-in-time.** Scores are updated periodically, not in real time. A project may improve or decline between updates.

2. **Subjective.** Despite the rubric, scoring requires judgment. Two analysts might score the same project differently.

3. **No price prediction.** A high Returns Score does not mean the price will go up. Market dynamics, sentiment, and external factors dominate price action.

4. **Does not capture everything.** Team quality, competitive landscape, regulatory risk, and technical execution are covered in project reviews, not the score.

5. **Early-stage uncertainty.** Many DeAI projects are pre-revenue or pre-product. Scores reflect current state, not future potential.

---

## How to Use Returns Score

- **Filter, not decide.** Use scores to narrow options, not to make final decisions.
- **Read the reasoning.** The number is less important than why we scored it that way.
- **Compare, do not absolutize.** Grades are relative. A "B" may be the best available in a category.
- **Check the date.** Scores are updated as projects evolve. Old scores may not reflect current reality.
- **Both scores matter.** A high Returns Score with low Freedom Score may be profitable but centralized. A high Freedom Score with low Returns Score may be sovereign but economically unproven.

---

## Updating the Score

Returns Scores are updated when:

- Material protocol changes (new fee mechanisms, emission updates, governance decisions)
- Quarterly reviews (minimum)
- Significant market events (halvings, unlocks, listings)
- New information becoming available

## Relationship to Freedom Score

The two scores are independent:

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│     HIGH RETURNS                                            │
│          │             ┌─────────────┐                     │
│          │             │ Quadrant A  │                     │
│          │    C        │ Ideal but   │                     │
│   (Render, Venice)     │ rare        │                     │
│          │             └─────────────┘                     │
│ LOW      ├───────────────────────────── HIGH FREEDOM       │
│ FREEDOM  │             ┌─────────────┐                     │
│          │             │ Quadrant B  │                     │
│          │    D        │ (Morpheus,  │                     │
│          │ (Grass,     │ Bittensor,  │                     │
│          │  io.net)    │ Akash)      │                     │
│          │             └─────────────┘                     │
│          LOW RETURNS                                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**Quadrant A (High Freedom, High Returns):** The ideal. Does not meaningfully exist in DeAI yet. Build it or find it.

**Quadrant B (High Freedom, Low Returns):** Sovereignty plays. Early-stage networks with genuine decentralization but unproven economics. Higher risk, higher conviction required.

**Quadrant C (Low Freedom, High Returns):** Profitable but centralized. Functional businesses with tokens attached. May generate returns but do not advance sovereignty.

**Quadrant D (Low Freedom, Low Returns):** Avoid. Neither decentralized nor economically viable.
