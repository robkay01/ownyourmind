## What it does

Morpheus is building a peer-to-peer network for AI agents. Think of it as the infrastructure layer that lets you run a personal AI assistant that isn't controlled by any company — it can access decentralised compute, connect to smart contracts, and operate autonomously on your behalf.

The network connects four key participants: compute providers, code contributors, capital providers, and community builders. Each group earns MOR tokens for their contributions.

## Value proposition

Unlike centralised AI assistants (Siri, Alexa, ChatGPT), a Morpheus agent runs on infrastructure you choose, uses models you select, and follows instructions only you give it. There's no company in the middle deciding what your agent can or can't do.

The practical upside: an AI agent that can interact with DeFi protocols, manage wallets, and execute complex multi-step tasks without trusting a centralised intermediary.

## Tokenomics

MOR follows a fair launch model — no pre-mine, no VC allocation. Tokens are distributed daily to the four contributor groups based on a fixed emission schedule. Capital providers stake stETH and direct yield to the protocol; in return they receive MOR emissions proportional to their stake.

The emission curve is designed to front-load early participants while maintaining long-term sustainability. See our [detailed tokenomics analysis](/tokenomics/how-mor-actually-works/) for the full breakdown.

## How to participate

- **Provide compute**: Run a Morpheus node and serve inference requests
- **Contribute code**: Submit pull requests to the Morpheus GitHub repositories
- **Provide capital**: Stake stETH through the Morpheus smart contracts
- **Build community**: Create content, documentation, or tooling

## Honest assessment

**Strengths**: Genuine fair launch, strong technical team, clear vision for agent-based AI, active development with regular releases.

**Weaknesses**: Agent capabilities still early-stage, network effects haven't kicked in yet, competing with well-funded centralised alternatives.

**Risk**: The agent use case needs to prove real utility beyond crypto-native tasks to achieve mainstream adoption.

## Freedom score: 8/10

High marks for fair launch tokenomics, open-source code, and genuine decentralisation of compute. Loses points because the smart contract layer still has some centralised control mechanisms during the bootstrap phase.
