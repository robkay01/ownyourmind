# Episode 015 — ElizaOS: Great Framework, Broken Token
## Turn-by-Turn Production Script

## TURN 01 — BOBSKI
```
Welcome back to the Own Your Mind podcast. I'm Bobski, and today we're covering ElizaOS -- the most popular open-source AI agent framework in web3. 17,662 GitHub stars. 583 contributors. 90-plus plugins. MIT licensed. Three minutes from install to a running agent.

But the token has collapsed from its effective all-time high. No staking, no fee accrual, no burns, no functioning governance vote. A middling Freedom Score and one of the lowest Returns Scores we have given. A great framework bolted onto a broken token economy.
```

## TURN 02 — RACHEL
```
And I'm Rachel. ElizaOS is the clearest example in DeAI of excellent open-source software that has no business having a token.
```

## TURN 03 — BOBSKI
```
Give us the origin story.
```

## TURN 04 — RACHEL
```
Launched in October 2024 as "ai16z" on Solana -- a provocative play on Andreessen Horowitz's name. The concept: an AI-managed DAO where an agent called Marc AIndreessen would autonomously run a venture fund. Initial token sale on DAOs.fun raised 420.69 SOL. In January 2025, Chris Dixon from a16z requested a name change, and it rebranded to ElizaOS. Created by Shaw Walters, who previously built the Webaverse game platform, which collapsed after a treasury security breach.
```

## TURN 05 — BOBSKI
```
What does the framework actually do?
```

## TURN 06 — RACHEL
```
It's a TypeScript framework for building autonomous AI agents. You define your agent's personality via a character file, plug in any LLM -- OpenAI, Anthropic, Google, or Ollama for local inference -- connect it to platforms like Discord, Telegram, and X, and deploy. The plugin system covers blockchain integrations, social media bots, RAG knowledge bases. Multi-agent coordination through Worlds and Rooms. auto.fun adds a no-code path -- create and deploy agents via a web interface with token launches on Raydium.
```

## TURN 07 — BOBSKI
```
The developer adoption is real.
```

## TURN 08 — RACHEL
```
Unquestionably. 17,662 stars, 5,439 forks, 583 contributors. An arXiv paper with 14 co-authors. Stanford's FDCI partnership with a grant exceeding 250,000 dollars for AI agent research. These metrics are genuine and verifiable. The framework itself is near-best-in-class.
```

## TURN 09 — BOBSKI
```
But the token.
```

## TURN 10 — RACHEL
```
The original AI16Z token reached its peak before the migration. Adjusted for the swap ratio, the current price represents a near-total decline. The "first AI VC fund" thesis hasn't generated demonstrable returns. The DAO treasury value is unverified.
```

## TURN 11 — BOBSKI
```
And the migration changed the distribution story.
```

## TURN 12 — RACHEL
```
Completely. Chapter one was a genuine fair launch -- no pre-mine, no VC allocation. Chapter two: the November 2025 migration expanded supply by 40 percent, from 6.6 billion to 11 billion tokens, creating new allocations for SAFT investors at 15 percent, team at 10 percent, foundation at 4.5 percent, ecosystem at 2.5 percent. These allocations didn't exist in the original fair launch. Original holders were diluted without formal on-chain governance approval.
```

## TURN 13 — BOBSKI
```
The Freedom Score is middling. What stands out?
```

## TURN 14 — RACHEL
```
Open source transparency scores near-best-in-class. Dozens of public repos, MIT licence, active development. Censorship resistance scores well because the framework is fully open source and can be self-hosted with local models via Ollama. Data sovereignty also scores well -- self-hosted agents store all data locally.
```

## TURN 15 — BOBSKI
```
Where does it fall short?
```

## TURN 16 — RACHEL
```
Infrastructure decentralisation scores poorly because there is no decentralised infrastructure layer -- no node network, no compute marketplace. The DAO treasury is managed by a single AI agent. Governance also scores poorly. The AI agent Marc AIndreessen retains final decision authority over treasury investments. The 40 percent supply expansion, the rebranding, the SAFT deals -- all decided unilaterally by Eliza Labs. There's a governance voting module promised but not shipped.
```

## TURN 17 — BOBSKI
```
There's also a security vulnerability.
```

## TURN 18 — RACHEL
```
Princeton researchers demonstrated that attackers can inject false context into an agent's persistent memory via social media interactions, manipulating trading decisions through Sybil attacks. Eliza Labs' response: quote, "no direct updates were made as a result." For agents potentially handling financial transactions, an acknowledged but unpatched vulnerability is concerning.
```

## TURN 19 — BOBSKI
```
The Returns Score is an F grade -- one of the lowest we have scored. Break it down.
```

## TURN 20 — RACHEL
```
Revenue sustainability scores near the bottom. There is no revenue model for the token. The framework is open source and free. auto.fun generates some fees but the connection to token holders is undefined. Token utility scores poorly -- no staking, no fee accrual, no burns, governance voting not live. You can use the entire ElizaOS framework without ever touching the token. Value accrual scores very poorly. There is no mechanism by which framework adoption drives token value. Brilliant open-source adoption with zero value capture.
```

## TURN 21 — BOBSKI
```
The honest verdict?
```

## TURN 22 — RACHEL
```
The framework deserves respect. The token demands scepticism. A small market cap for the most popular AI agent framework in web3 suggests the market has concluded that being great open-source software doesn't translate into token value. And the market is probably right. The potential L1 blockchain with Saga would change the equation if it ships, but plans aren't products. For now, the framework is worth using. The token is not worth holding.
```
