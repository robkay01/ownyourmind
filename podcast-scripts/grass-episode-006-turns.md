# Episode 006 — Grass: Eight Million Nodes and Zero Public Repos
## Turn-by-Turn Production Script

## TURN 01 — BOBSKI
```
Welcome back to the Own Your Mind podcast. I'm Bobski, and today we're covering Grass -- a bandwidth-sharing network on Solana where users install a browser extension and contribute unused internet bandwidth to scrape web data for AI training. Millions of monthly active nodes across 190 countries.

A low Freedom Score and a middling Returns Score. Those tell a very different story from the headline metrics.
```

## TURN 02 — RACHEL
```
And I'm Rachel. Grass is a fascinating case because the scale is genuinely impressive. But scale and decentralisation are not the same thing. Millions of nodes doesn't make a network decentralised if a single company controls all the code, all the revenue, and all the data.
```

## TURN 03 — BOBSKI
```
Walk us through what Grass actually does.
```

## TURN 04 — RACHEL
```
You install a browser extension, desktop app, or Android app. It contributes your unused internet bandwidth to scrape public web data. AI companies need vast quantities of training data, and millions of residential IP addresses provide better coverage than data centre scraping because residential IPs bypass bot detection and geo-restrictions.

The enterprise product is called Live Context Retrieval -- when an AI model gets a prompt, Grass retrieves relevant current information from the web before the model responds. Every LCR call requires GRASS tokens.
```

## TURN 05 — BOBSKI
```
The scale numbers are real?
```

## TURN 06 — RACHEL
```
They are. Millions of monthly active nodes, over a million concurrent connections, petabytes of data retrieved daily. Enterprise customers described as including the largest AI labs, with seven-figure revenue clients. The VALID dataset collaboration produced terabytes of multimodal data. CoinGecko, Nansen, and the project's own metrics align on these figures.
```

## TURN 07 — BOBSKI
```
Freedom Score is an F grade -- one of the lowest we have covered. Why so low with millions of nodes?
```

## TURN 08 — RACHEL
```
Because the nodes are just bandwidth endpoints. The core is entirely centralised.

Open source transparency scores very poorly -- and here's why. Wynd Network's GitHub has zero public repositories. Not some. Zero. Node software, router software, validator software, ZK processor -- all proprietary. You cannot independently verify a single technical claim.

Governance scores near the bottom too. The docs reference DAO governance, but no proposals have been submitted, no votes have occurred, no governance forum exists. All decisions are made by Wynd Labs and the Grass Foundation.
```

## TURN 09 — BOBSKI
```
Data sovereignty?
```

## TURN 10 — RACHEL
```
Scores poorly. Users contribute bandwidth but don't own or control the scraped data. The Foundation captures all value and sells it to enterprise clients. You have no control over what's scraped through your connection, no visibility into who buys the data, no say in pricing. Your residential IP address is being used as an exit node for web scraping. If a client scrapes restricted content through your connection, the legal exposure sits with you, not the Foundation.
```

## TURN 11 — BOBSKI
```
That exit node liability -- how serious is that risk?
```

## TURN 12 — RACHEL
```
It's the defining risk for participants. The legal landscape for web scraping is shifting rapidly. The EU Digital Services Act, GDPR enforcement, and US CFAA precedents are all creating a more restrictive environment. Despite client vetting and AppEsteem cybersecurity certification, if regulators decide bandwidth contributors are facilitating unauthorised data collection, millions of users face potential exposure.
```

## TURN 13 — BOBSKI
```
Returns Score -- middling, a C grade. What's the token picture?
```

## TURN 14 — RACHEL
```
The insider allocation is 47.2 percent -- 25.2 percent to investors, 22 percent to team. The Foundation controls another 22.8 percent. Community allocation is 30 percent. A major cliff released a large tranche of investor tokens, and monthly unlocks continue for years. The token is deep below its all-time high.

Revenue sustainability is actually the strongest dimension -- real enterprise customers paying for real data. But value accrual is weak because all revenue flows through the Foundation with zero transparency. No figures published, no on-chain verification of enterprise payments.
```

## TURN 15 — BOBSKI
```
No burn mechanism either?
```

## TURN 16 — RACHEL
```
No burn, no fee distribution, no verifiable buyback programme. Token holders are asked to trust that value accrues, but the mechanism connecting Foundation revenue to GRASS demand is entirely opaque.
```

## TURN 17 — BOBSKI
```
The airdrop was notable though.
```

## TURN 18 — RACHEL
```
It was. 2.8 million recipients received 10 percent of supply with no lockup -- one of the widest distributions in Solana history. The 500-point minimum threshold meant anyone who ran the extension for roughly 100 hours could qualify. Credit where it's due. But a wide airdrop does not offset 47.2 percent insider concentration.
```

## TURN 19 — BOBSKI
```
The honest verdict?
```

## TURN 20 — RACHEL
```
The business works. The enterprise traction is real. The scale is genuine. But this is a centralised data business with a token incentive layer. The code is entirely closed-source, governance doesn't exist in practice, users have no sovereignty over the data their bandwidth produces, and all revenue flows through an opaque Foundation. The widest airdrop in Solana history doesn't make a network decentralised.
```
