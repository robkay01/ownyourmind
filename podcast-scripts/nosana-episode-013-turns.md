# Episode 013 — Nosana: Inference Bet With Zero Visibility

But zero public revenue data. Dangerously thin liquidity. And the vast majority of registered hosts sitting idle on any given day. A middling Freedom Score and a weak Returns Score. Both need explaining.

## TURN 02 — RACHEL

## TURN 03 — BOBSKI
```
What's the inference thesis?
```

## TURN 04 — RACHEL
```
Most decentralised compute projects try to be everything -- training, inference, rendering, general compute. Nosana narrows the focus to inference only. That's the workload that scales with AI adoption. Every chatbot query, every image generation, every voice transcription is an inference job. As AI moves from research to production, inference demand grows exponentially while training remains periodic. The pitch is up to 6x cost savings versus AWS, Azure, and GCP.
```

## TURN 05 — BOBSKI
```
And the team?
```

## TURN 06 — RACHEL
```
Founded in Amsterdam in 2021 by Jesse Eisses -- MSc in Artificial Intelligence from the University of Amsterdam, former ML Platform Architect at Intel -- and Sjoerd Dijkstra, a senior DevOps engineer. Originally launched as a decentralised CI/CD platform for CPU workloads and pivoted to GPU-based AI inference in October 2023. Total funding: only 1.74 million dollars. That's one of the most modestly funded projects in DeAI.
```

## TURN 07 — BOBSKI
```
The network has real traction though?
```

## TURN 08 — RACHEL
```
Over 2 million deployments by August 2025 -- 150 percent increase from 800,000 in Q1. The Render Network integration, approved January 2025, allows Render nodes to execute Nosana jobs. 14 named partnerships. Supports LLM inference, Stable Diffusion, Whisper speech-to-text, and arbitrary Docker containers. Mainnet launched January 2025 after a year-long closed beta.
```

## TURN 09 — BOBSKI
```
But the revenue?
```

## TURN 10 — RACHEL
```
Zero public figures. Two million deployments with no disclosure of revenue per deployment, total fees generated, or protocol profits. It's impossible to assess whether Nosana has found product-market fit or is giving away compute at unsustainable prices. Not on DeFiLlama. No independent verification of any kind.
```

## TURN 11 — BOBSKI
```
The Freedom Score is a C grade. What's working?
```

## TURN 12 — RACHEL
```
Infrastructure decentralisation scores well. Thousands of active nodes across 60-plus countries with permissionless GPU hosting -- no KYC required. Jobs matched via on-chain smart contracts on Solana. Censorship resistance also scores well. Anyone with a GPU can host without permission. Docker-based execution provides isolation.
```

## TURN 13 — BOBSKI
```
Where does it fall short?
```

## TURN 14 — RACHEL
```
Governance scores poorly. There's a governance framework called NNPs, but only one proposal has ever been submitted in four-plus years. Proposals require team review before community vote -- that's gatekeeping. No Foundation established yet. Token distribution fairness also scores below average because team plus company held 45 percent at launch, with only 8 percent initial public distribution through the IDO and airdrop.
```

## TURN 15 — BOBSKI
```
The Returns Score is a D grade. The token economics?
```

## TURN 16 — RACHEL
```
Token utility scores decently. NOS is required for GPU job payments. Staking via xNOS determines fee share. There's a buyback-and-burn mechanism where 20 percent of protocol profits buy and burn NOS. Good design on paper. But revenue sustainability scores near the bottom. No public revenue despite millions of deployments. Competing against larger rivals with zero disclosed revenue and minimal funding.
```

## TURN 17 — BOBSKI
```
And the liquidity situation?
```

## TURN 18 — RACHEL
```
Dangerously thin. Liquidity and access scores poorly. Gate.io, MEXC, and Raydium. No Tier 1 CEX. Low daily volume and shallow DEX liquidity. Any significant buy or sell causes major slippage. With a small market cap, position sizing must be very conservative.
```

## TURN 19 — BOBSKI
```
There's also the NNP-0001 proposal to watch.
```

## TURN 20 — RACHEL
```
Right. Voted in November 2025, it proposes shifting rewards from passive yield to usage-based emissions tied to network activity. If implemented, this could introduce inflation beyond the original 100-million-token cap -- fundamentally changing supply dynamics. On the positive side, all original vesting is complete. 83.4 percent circulating. FDV-to-market-cap ratio of 1.2x, which is healthy. No ongoing insider unlock pressure.
```

## TURN 21 — BOBSKI
```
The honest verdict?
```

## TURN 22 — RACHEL
```
The inference thesis is sound. The technology works. The Render integration is a credible supply expansion. But without revenue visibility, you're making a pure conviction bet. Two million deployments could mean thriving product-market fit, or it could mean free compute that generates no sustainable economics. The high host idle rate -- a small fraction of registered nodes are active daily -- suggests most capacity sits unused. This is Quadrant B: high freedom, low returns. A sovereignty play with unproven economics.
```
