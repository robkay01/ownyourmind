## TURN 03 — BOBSKI
```
How does the architecture work?
```

## TURN 04 — RACHEL
```
Three tiers. Enterprise Validator Nodes -- 20 to 30 permissioned nodes run by Google, Samsung, Sony, and others proposing and finalising blocks. Guardian Nodes -- approximately 3,500 permissionless community nodes that seal blocks and check the enterprise validators. Think of them as the accountability layer. And Edge Nodes -- 10,000 plus nodes providing compute, storage, and video relay, collectively delivering about 80 petaflops of distributed GPU compute.
```

## TURN 05 — BOBSKI
```
And the dual-token system -- explain why this matters.
```

## TURN 06 — RACHEL
```
THETA is governance and staking only. It is not the gas token. TFUEL handles all payments, transaction fees, and smart contract interactions. TFUEL has about 5 percent annual inflation for staking rewards and a burning mechanism where a minimum 25 percent of each TFUEL payment to edge nodes is burned. So if you hold THETA, you are staking to earn inflationary TFUEL rewards -- not capturing protocol revenue directly.
```

## TURN 07 — BOBSKI
```
So the value flows to TFUEL?
```

## TURN 08 — RACHEL
```
Correct. EdgeCloud revenue goes to Theta Labs Inc, not to THETA token holders. There is no buyback, no revenue share, no direct fee capture mechanism for THETA. THETA's value rests entirely on staking demand to secure the network and speculative premium from the enterprise partnerships. That is a thin basis for long-term value capture.
```

But validators are permissioned. Only enterprises with a large THETA stake can validate. Primary block production is concentrated among a small number of entities.

The 36.3 percent Theta Labs reserve concentrates influence further.
