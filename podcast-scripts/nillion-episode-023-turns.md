## TURN 03 — BOBSKI

```
How does blind computation actually work?
```

## TURN 04 — RACHEL
```
Three privacy-enhancing technologies combined. Multi-party computation splits data across nodes -- each holds a meaningless fragment. Homomorphic encryption allows mathematical operations on encrypted data. Trusted Execution Environments provide hardware-isolated enclaves even the operator cannot inspect.

The network has a dual-layer architecture. nilChain is a Cosmos-based coordination layer for payments, staking, and governance. Petnet is where the blind computation actually happens, with three modules -- nilDB for encrypted storage, nilVM for blind computation, and nilAI for private LLM inference inside TEEs.
```

```
The counter-argument -- blind computation is slow and expensive.
```

## TURN 16 — RACHEL
```
Correct. MPC requires multiple rounds of communication between nodes. Homomorphic encryption adds orders-of-magnitude computational overhead. TEEs depend on hardware vendor trust -- Intel SGX has had side-channel attacks. The question is not whether privacy-preserving computation is desirable. It obviously is. The question is whether Nillion can make it fast and cheap enough for production AI workloads. That has not been demonstrated.
```
