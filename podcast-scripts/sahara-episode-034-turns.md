# Episode 034 — Sahara AI: A Centralised Company in Decentralised Clothing
## Turn-by-Turn Production Script

## TURN 01 — BOBSKI
```
Welcome back to the Own Your Mind podcast. I'm Bobski, and today we're covering Sahara AI -- a VC-backed AI platform with 51.5 million in funding, a USC professor as CEO, and a real data labelling business. But the blockchain is not live, the token contract has a pause button, and SAHARA is deep below its all-time high.

A low Freedom Score and a weak Returns Score. Let's break it down.
```

## TURN 02 — RACHEL
```
And I'm Rachel. Sahara is what happens when a well-funded centralised AI company wraps itself in decentralised branding. The data labelling is real. The decentralisation is not.
```

## TURN 03 — BOBSKI
```
Who is behind Sahara?
```

## TURN 04 — RACHEL
```
Sean Ren is the CEO -- an Associate Professor of Computer Science at USC, holder of the Viterbi Early Career Chair, MIT Technology Review Innovator Under 35, and he directs the INK Research Lab at USC while contributing to the Allen Institute for AI. Tyler Zhou is COO, formerly Investment Director at Binance Labs and Executive Director at Wells Fargo. The entity is registered in the Cayman Islands with a team of roughly 92 people.
```

## TURN 05 — BOBSKI
```
What are they building?
```

## TURN 06 — RACHEL
```
The vision is a full-stack AI-native blockchain covering data labelling, model training, an AI marketplace, and a dedicated Layer 1 called Sahara Chain. What actually exists today is a data labelling platform. Season 2 processed 2.55 million approved data points at 95 percent accuracy with over 200,000 users. That's comparable to what Scale AI and Appen deliver in the centralised world.
```

## TURN 07 — BOBSKI
```
And the blockchain?
```

## TURN 08 — RACHEL
```
Not operational. Sahara Chain mainnet was targeted for Q3 2025. It is registered on ChainList with a chain ID and RPC endpoint, but there is no public block explorer, and multiple sources describe the launch as imminent as of March 2026. That's nine months past the target date. For a project that calls itself the world's first full-stack AI-native blockchain, having no functioning blockchain is a fundamental credibility gap.
```

## TURN 09 — BOBSKI
```
The Freedom Score is an F grade. That's one of the lowest we've covered.
```

## TURN 10 — RACHEL
```
Infrastructure decentralisation scores very poorly -- the chain is not live and validators are permissioned, limited to approved professional operators. Governance also scores very poorly -- no proposals submitted, no voting, all decisions made by the founding team. Censorship resistance scores poorly, and here is the kicker -- the token contract includes ERC20Pausable functionality. The contract owner can halt all SAHARA token transfers at any time. For a project promoting AI sovereignty, a pause button is a remarkable contradiction.
```

## TURN 11 — BOBSKI
```
A pause function. So someone can just stop all token transfers?
```

## TURN 12 — RACHEL
```
Correct. The contract owner can freeze the token. No timelock, no governance vote required. That single feature undermines every decentralisation claim the project makes. Compare that with projects like Vana, which has nine published security audits and a live mainnet with 1.3 million users.
```

## TURN 13 — BOBSKI
```
The VC backing is substantial though.
```

## TURN 14 — RACHEL
```
51.5 million across three rounds. Polychain Capital led the seed. Pantera, Polychain, and Binance Labs co-led the Series A. Sequoia Capital, Samsung NEXT, and Matrix Partners also participated. And the exchange coverage is excellent -- Binance, OKX, KuCoin, Bitget, MEXC, Upbit, Bithumb. Strong daily volume. The distribution infrastructure is top tier.
```

## TURN 15 — BOBSKI
```
The Returns Score is weak. What drives it?
```

## TURN 16 — RACHEL
```
Revenue sustainability is notable for the right reasons -- the data labelling business is real and growing, improving in accuracy between seasons. But none of that revenue flows through the token or the blockchain. Token utility scores middling -- planned for Sahara Chain but currently just a standard ERC-20 on Ethereum and BNB Chain. Value accrual scores poorly because the connection between enterprise data labelling revenue and token value is aspirational.
```

That's the critical date. Over a billion tokens -- over 10 percent of total supply -- begin unlocking when the investor and team cliff expires. A significant amount of insider tokens becoming liquid, hitting a token already deep below its all-time high. The remaining 75 percent of insider tokens unlock linearly over the following 36 months.

## TURN 19 — BOBSKI
```
The GitHub metrics tell an interesting story too.
```

## TURN 20 — RACHEL
```
The primary chain repository has 128 commits, zero stars, and zero forks. Compare that to 913,000 X followers and 342,000 Discord members. The community is there for the airdrop, not the code. No published security audits exist despite 51.5 million in funding. That is indefensible at this stage.
```

## TURN 21 — BOBSKI
```
Honest verdict?
```

## TURN 22 — RACHEL
```
Sean Ren's credentials are genuinely exceptional and the data labelling traction is real. But this is a well-funded centralised AI company, not a decentralised network. The blockchain is not live, governance does not exist, validators are permissioned, and the token has a kill switch. I would reconsider if the mainnet launches with permissionless validators, an independent security audit is published, and the pause function is removed or governance-locked.
```
