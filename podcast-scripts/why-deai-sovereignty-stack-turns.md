
## TURN 03 — BOBSKI
```
So the sovereignty stack has five layers. Walk me through them from the bottom up.
```

## TURN 04 — RACHEL
```
Bottom to top: compute, data, models, inference, and coordination. Compute is where your models run. Data is your training data sources. Models are the open weights you can run and modify. Inference is who sees your prompts. And coordination is how distributed participants align incentives. No single protocol solves all five. Sovereignty requires composition.
```

## TURN 05 — BOBSKI
```
Let's start with compute. What are the options?
```

## TURN 06 — RACHEL
```
Akash is the most credible. Mainnet since September 2020. Real, verifiable on-chain revenue and growing year-over-year. H100s at a significant discount to AWS. No KYC, reverse auction model where providers compete on price. But the active provider count is small. That's not a cloud marketplace yet. That's a pilot programme.
```

## TURN 07 — BOBSKI
```
What about Render and io.net?
```

## TURN 08 — RACHEL
```
Render has thousands of node operators and genuine enterprise customers -- Hollywood studios, Apple Vision Pro projects. But it's permissioned. You need approval to run a node. Core is closed-source. It scores poorly on freedom. io.net markets hundreds of thousands of "registered" GPUs but the daily average of verified active GPUs is a tiny fraction of that. After the 2024 Sybil attack that inflated GPU counts, every metric deserves scrutiny.
```

## TURN 09 — BOBSKI
```
Moving up the stack -- data sovereignty. This is the hard one.
```

## TURN 10 — RACHEL
```
The hardest. Grass has millions of monthly active nodes, but users contribute bandwidth, not data ownership. They don't own the data, don't share in revenue, and bear legal risk as exit nodes. That's a bandwidth marketplace, not data sovereignty. Vana is different -- users export personal data and contribute to Data DAOs with TEE-based verification and user-controlled keys. That's genuine data sovereignty. Ocean Protocol has the best technology in Compute-to-Data, but the organisation is in governance crisis with active litigation.
```

## TURN 11 — BOBSKI
```
The model layer -- this is where things have gotten exciting recently.
```

## TURN 12 — RACHEL
```
Open models in 2026 are legitimately competitive. Llama 4 Scout, Qwen 3.5, DeepSeek V3 -- you can download, inspect, modify, and run these. And QLoRA changed the fine-tuning game completely. A 70 billion parameter model is now trainable on a single consumer GPU with 48 gig of VRAM. 10 to 24 dollars per run versus 240 to 360 for full fine-tuning. That's a genuine sovereignty enabler.
```

## TURN 13 — BOBSKI
```
Inference privacy -- we covered this in the privacy episode, but how does it fit the stack?
```

## TURN 14 — RACHEL
```
Three tiers. Highest sovereignty is local inference -- no third party, but hardware limits. Next is Phala with TEE-based inference, hardware-enforced privacy with remote attestation. Then Venice -- no logging, but GPU providers see plaintext prompts. The trade-off is always between privacy guarantees and model capability.
```

## TURN 15 — BOBSKI
```
And the coordination layer at the top?
```

## TURN 16 — RACHEL
```
Bit-tensor is the largest -- dozens of subnets, real workloads. But centralised block production and the stake-weight problem where wealth determines earnings more than AI quality. Morpheus is the only genuine fair launch -- every "More" token earned through contribution. But thin liquidity -- meaningful positions are hard to enter or exit without slippage.
```

## TURN 17 — BOBSKI
```
Alright, the practical question everyone wants answered -- what does this actually cost?
```

## TURN 18 — RACHEL
```
Clear break-even point: roughly 2 million tokens per day. Below that, APIs are cheaper. Above that, self-hosting becomes economical. An entry-level individual stack -- consumer GPU, Llama 8 billion, Ollama for local inference -- costs zero to 200 a month with high sovereignty but limited model size. A professional stack for a small team -- Akash H100, Qwen 3.5, Phala for TEE inference -- runs 2 to 4 thousand a month. Enterprise with 400 billion plus models, 40 to 100 thousand a month.
```

## TURN 19 — BOBSKI
```
And the hybrid approach?
```

## TURN 20 — RACHEL
```
A fintech case study from the research -- pure OpenAI API cost 47 thousand a month. Hybrid with self-hosted open-source models, 8 thousand. That's 83 percent savings. You don't need full sovereignty everywhere. Route non-sensitive queries to cheaper APIs. Keep sensitive workloads on infrastructure you control.
```
